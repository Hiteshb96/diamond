Compile & Link

To compile and link Diamond a project Makefile must be generated by Autotools. This is step one shown below. This step is only required if you check out the Diamond source.

The Diamond Makefile.am is included in the source packages.

Note: All of the Autotool commands are run from a Bash prompt. For information about installing a Bourne shell (Bash) on Windows, refer to Building for Windows.

(1)  Run Autoconf from your project directory with the syntax shown below. This step is run once or anytime configure.ac is modified. When this program runs successfully you will see no output.

autoreconf -i


(2)  Run configure from your project directory with the syntax shown below. This step is run only one time.

./configure


(3)  To build Diamond from your project directory use the syntax shown below. This step is run every time the Diamond source is modified.

make

To build Diamond and deploy to the location specified in your bin/Makefile.am, use the following syntax.

make install

This following command is run from your project directory. It will delete all of the object and intermediate files in the 'bin' folder. The next time you run 'make' every file will be compiled, since they were deleted.

make clean
